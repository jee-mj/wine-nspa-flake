# Linux 7.1.2 NSPA kernel migration

## Goal

Move the NSPA kernel package from Linux 7.0.12 to Linux 7.1.2 while retaining
PREEMPT_RT, the existing NSPA ntsync priority-inheritance patches, and the
existing NixOS integration.

## Scope

- Update `flake.nix` in this repository to fetch Linux 7.1.2.
- Replace the existing 7.0.1-rt2 patch with kernel.org's 7.1.1-rt2 patch.
- Preserve NSPA patches `1001` through `1015`, the NSPA kernel config, and the
  exported `linux-nspaPackages` interface.
- Add a focused Nix evaluation/build check for the kernel package.
- Build and validate the kernel package before deployment by a downstream
  configuration.

## Design

The package continues to use `pkgs.buildLinux` and `pkgs.linuxPackagesFor`.
Its source changes to the kernel.org Linux 7.1.2 tarball, with version and
module directory version aligned to the new source and RT suffix. The RT patch
is applied first, followed by the unchanged NSPA patch series. This preserves
the existing patch layering: generic realtime changes precede NSPA's ntsync
priority-inheritance changes.

The exported `linux-nspaPackages` interface remains unchanged for downstream
consumers. Existing NixOS generations remain available through the boot loader
and are the rollback mechanism if the new kernel fails to boot or regressions
are found.

## Verification

1. Check patch applicability and build the NSPA kernel derivation.
2. Evaluate a downstream NixOS configuration that consumes the package.
3. Commit the package migration.
4. Activate it from the downstream configuration.
5. Reboot, then confirm:
   - `uname -r` identifies the new 7.1.2 RT kernel;
   - `/sys/kernel/realtime` reports realtime enabled;
   - the `ntsync` module/device is available;
   - NVIDIA loads and the graphical session starts;
   - PipeWire/JACK audio services are active.

If a patch fails, the kernel does not build, or a runtime check fails, do not
continue deployment. Reboot the previous NixOS generation and retain its
kernel package as the rollback path.

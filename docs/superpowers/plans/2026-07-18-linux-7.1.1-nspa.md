# Linux 7.1.1 NSPA Kernel Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Produce a reproducibly checked Linux 7.1.1 NSPA kernel using the exact `7.1.1-rt2` PREEMPT_RT patch and a semantically ported NSPA patch series.

**Architecture:** `flake.nix` remains the sole package-definition boundary: `pkgs.buildLinux` fetches the exact kernel source, applies the exact RT patch first, then the reviewed NSPA sequence, and exposes the result through `linux-nspaPackages`. A flake check builds `linux-nspaPackages.kernel`; a downstream toplevel build remains the stronger integration gate for NVIDIA and external modules.

**Tech Stack:** Nix flakes, `pkgs.buildLinux`, `pkgs.linuxPackagesFor`, kernel.org Linux and PREEMPT_RT archives, NixOS kernel module build integration.

## Global Constraints

- Use Linux 7.1.1 with exact `patch-7.1.1-rt2`; do not use Linux 7.1.2, Linux 7.1.3, or forward-port the PREEMPT_RT patch.
- The NSPA series is a semantic forward port from Linux 7.0.12 to Linux 7.1.1; application alone is insufficient proof.
- Preserve this exact NSPA order: `1001`, `1002`, `1003`, `1004`, `1007`, `1009`, `1010`, `1011`, `1012`, `1013`, `1014`, `1015`.
- Record every source/patch hash and every patch offset, fuzz, reject, already-upstream hunk, or manual rebase.
- No package build or implementation commit may proceed while any NSPA patch
  evidence row is `pending-task-2`, incomplete, or unresolved.
- Require final `CONFIG_PREEMPT_RT=y`, `CONFIG_NTSYNC=y` or `CONFIG_NTSYNC=m`, `CONFIG_HIGH_RES_TIMERS=y`, the existing timer frequency, existing `NO_HZ` mode, and all existing NSPA-critical options.
- Stop after package changes, checks, kernel/downstream builds, and commit. Do not activate, reboot, or deploy.

contains `pending-task-2`, an incomplete field, or an unresolved result.
